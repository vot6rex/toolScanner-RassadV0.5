# Contributing

Thanks for your interest in contributing to Analyzer PRO!

## Before contributing
- Read the repository README and the legal/ethics notice.
- Do **not** test code on systems or networks without explicit permission.

## Reporting bugs / feature requests
- Open an Issue describing:
  - Steps to reproduce (if a bug).
  - Expected vs actual behavior.
  - Relevant environment (OS, Python version, optional PyQt5 installed?).

## Pull requests
- Fork the repo, create a feature branch, and open a PR.
- Keep changes focused and documented.
- Include tests or manual verification steps for important features.
- Update README if you add or change user-facing behaviors.

## Code style
- Keep code readable and commented.
- Prefer clarity over cleverness.
